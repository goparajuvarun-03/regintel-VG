# RegIntel-AI · Deployment Guide

Step-by-step deploy to Streamlit Community Cloud. Total time: ~10 minutes.

## Prerequisites

- A GitHub account (free)
- This repo pushed to GitHub
- A Groq account (free, no credit card) OR a Gemini API key

## Step 1 — Push the repo to GitHub

```bash
cd regintel-ai
git init
git add .
git commit -m "Initial RegIntel-AI commit"
git branch -M main
git remote add origin https://github.com/YOURUSERNAME/regintel-ai.git
git push -u origin main
```

Confirm `.streamlit/secrets.toml` is **not** committed (the `.gitignore` should block it). Run `git status` and verify only `.streamlit/config.toml` and `.streamlit/secrets.toml.example` are tracked.

## Step 2 — Get an LLM API key

### Option A — Groq (recommended for demos: free, fast, generous limits)

1. Go to https://console.groq.com
2. Sign in with Google or email — no credit card needed
3. Click "API Keys" → "Create API Key"
4. Copy the key (starts with `gsk_...`) — save it somewhere safe; you can't view it again

**Free tier limits** (`llama-3.3-70b-versatile`):
- 30 requests/minute
- 1,000 requests/day
- 6,000 tokens/minute

More than enough for a demo. If you need more headroom, switch model to `llama-3.1-8b-instant` (14,400 RPD).

### Option B — Gemini

1. Go to https://aistudio.google.com/apikey
2. Click "Create API key"
3. Copy the key (starts with `AIza...`)

**Free tier** of Gemini 2.5 Flash-Lite is very limited (~1,000 RPD). If you hit the wall during a demo, switch to Groq via `LLM_PROVIDER = "groq"` in secrets.

## Step 3 — Create a GitHub personal access token (optional but recommended)

This enables cloud-sync — your data persists across Streamlit Cloud container restarts.

1. Go to https://github.com/settings/personal-access-tokens
2. Click "Generate new token" → "Fine-grained tokens"
3. Token name: `regintel-ai-sync`
4. Expiration: 90 days
5. Repository access: only the `regintel-ai` repo (don't grant all-repo access)
6. Permissions → Repository permissions → Contents → Read and write
7. Click "Generate" — copy the token (starts with `github_pat_...`)

Without this token, the app still works, but if Streamlit Cloud restarts your container you'll lose any uploaded documents.

## Step 4 — Deploy to Streamlit Cloud

1. Go to https://share.streamlit.io
2. Click "New app"
3. Repository: `YOURUSERNAME/regintel-ai`
4. Branch: `main`
5. Main file path: `streamlit_app.py`
6. App URL (optional): pick a slug like `regintel-demo`
7. Click "Advanced settings" → paste this into the Secrets box:

```toml
LLM_PROVIDER = "groq"

GROQ_API_KEY = "gsk_PASTE_YOURS_HERE"
GROQ_MODEL = "llama-3.3-70b-versatile"

# Optional — keeps data persistent across restarts
GITHUB_TOKEN = "github_pat_PASTE_YOURS_HERE"
GITHUB_REPO = "YOURUSERNAME/regintel-ai"
```

8. Click "Deploy!"

First deploy takes 2-3 minutes (Streamlit Cloud installs Python deps). Subsequent redeploys after a push are ~30 seconds.

## Step 5 — Verify everything works

When the app loads:

1. **Sidebar** should show `VG Health Plan · Compliance Workspace`
2. **Dashboard** should show 7 documents loaded (3 regs + 4 internal artifacts)
3. Open **Impact Analysis** → pick `CMS-4201-F Continuity of Care (Final)` → click "Run impact analysis"
4. You should see 2-3 impacted artifacts with priority badges and confidence scores
5. Click "Get proposed text" on any artifact → you should see a **green** banner: "✓ Generated by Groq (llama-3.3-70b-versatile)"

If the LLM-source badge shows red ("Gemini/Groq failed — fell back to mock"), check:
- `LLM_PROVIDER` is set correctly in secrets
- The API key is valid and not rate-limited
- Click the error message in the badge — it explains exactly what went wrong

## Step 6 — Pre-warm before live demos

Streamlit Cloud puts apps to sleep after inactivity. Cold-start can take 30+ seconds.

Before any demo:
1. Open the app URL 5 minutes ahead of time
2. Click through one page (Dashboard → Impact Analysis) to wake the container
3. Leave the tab open until showtime

## Troubleshooting

### "Module not found" errors
Push fixes the requirements.txt or `streamlit_app.py` and the auto-redeploy should pick it up. If not, click "Manage app" → "Reboot".

### App stuck on "Setting up..."
Common when the seed docs are large. Wait 60 seconds. If still stuck, check the deploy logs from the app dashboard.

### "Gemini rate limit exceeded"
You hit the free-tier quota. Switch to Groq:
- Streamlit Cloud → Settings → Secrets
- Change `LLM_PROVIDER = "gemini"` to `LLM_PROVIDER = "groq"`
- Save → app auto-redeploys

### Data disappeared after a deploy
This means GitHub sync isn't configured. Either:
- Set `GITHUB_TOKEN` and `GITHUB_REPO` in secrets, or
- Re-upload the documents via the Upload page (the seed/ folder also auto-reloads on startup)

### Want to wipe the database and start fresh
- Streamlit Cloud → Manage app → Reboot. This rebuilds the container; SQLite is recreated from `seed/`.
- If cloud-sync is on, manually delete the `regintel-data-snapshots` branch on GitHub first.
